# Git Retrospective

git log から繰り返し失敗パターンを検出し、根本原因を分析してADRに記録する。

## 分析手順

### Step 1: データ収集

以下のコマンドでコミット履歴を収集する:

```bash
# fix/revert コミットを抽出（直近200件）
git log --oneline -200 | grep -iE 'fix:|revert|hotfix'

# 同じファイルへの連続修正を検出
git log --oneline -200 --diff-filter=M --name-only | awk '/^[a-f0-9]/{commit=$0; next} /\./{files[commit]=(files[commit] ? files[commit] "," : "") $0} END{for(c in files) print c, files[c]}' | sort

# 同じキーワードを含むfixコミットをグループ化
git log --oneline -200 | grep -iE 'fix:' | sed 's/^[a-f0-9]* //' | sort
```

### Step 2: パターン検出

以下の観点で繰り返しパターンを特定する:

1. **矛盾修正**: A を追加 → A を削除（例: 自動削除追加 → 自動削除やめる）
2. **段階的修正**: 同じ機能を3回以上修正（例: ポート固定 → ポート変更 → 動的ポート）
3. **同一ファイル集中**: 短期間に同じファイルを複数回修正
4. **revert**: 変更を戻しているコミット

### Step 3: 根本原因分析

各パターンについて以下を分析する:

- **なぜ1回で正しく実装できなかったか？**
  - 環境差異を考慮していなかった
  - 副作用を予測できなかった
  - 要件が不明確なまま実装した
  - 既存の制約を把握していなかった
- **再発防止に必要な知識は何か？**
- **CLAUDE.md やスキルに追加すべきルールはあるか？**

### Step 4: レポート出力

分析結果を以下の形式で報告する:

```
## 繰り返しパターン検出レポート

### パターン 1: [名前]
- コミット: [関連コミット一覧]
- 問題: [何が繰り返されていたか]
- 根本原因: [なぜ繰り返したか]
- 対策: [再発防止策]
- アクション: [具体的な実施項目]
```

### Step 5: ADR作成

重要な対策は `/adr` スキルを使ってADRとして記録する。
ADRに記録すべき基準:
- 同じ領域で3回以上修正が発生したパターン
- 矛盾する修正（追加→削除）が発生したパターン
- 今後のコード変更に影響する設計判断

### Step 6: ルール反映

分析結果に基づき、必要に応じて以下を更新する:
- `CLAUDE.md`: プロジェクト全体に影響するルール
- `.claude/skills/*/SKILL.md`: 特定領域のスキル
- `docs/adr/`: 意思決定の記録
