# コード品質改善・リファクタリング

手動実行時（`/inline-quality`）は、指定された範囲のコードを以下のルールに基づいてリファクタリングする。
feat/fix 時は、変更に関連するコードの品質をその場で維持する。

背景: ADR 0005 参照。SOLID/DRY一括リファクタリングが4回繰り返された（PR #16, #56, #80, #98）。

## 手動実行時の手順

1. 対象範囲を確認する（ユーザー指定 or プロジェクト全体）
2. 以下のルールに違反している箇所を洗い出す
3. 違反箇所を修正する
4. `make deploy` を実行する

## ルール

### 1. 単一責任

1つの関数・クラスが複数の責務を持たないこと。
責務が2つ以上あるなら分割する。

### 2. 重複禁止

同じロジックが複数箇所にあるなら共通化する。
コピペしてから「後で共通化」は禁止。

### 3. デッドコード即時削除

使われていないコードはその場で削除する。
「念のため残す」は禁止。

### 4. feat/fix PRにリファクタリングを含める

その機能実装に必要な範囲の構造改善は feat/fix PR に含めてよい。
「refactor/solid-dry」のような一括リファクタリングブランチを作らない。
