# 0001. deploy.shでワークツリーを自動削除しない

Date: 2026-03-08

## Status

Accepted

## Context

deploy.sh のワークツリー処理で5回の修正が発生した:

- `980c32a` deploy.shのワークツリー実行時にdevelop切替とサーバー再起動をメインリポジトリで行うように修正
- `3d23ca0` deploy.shからワークツリー削除を除去し、セッション終了時のクリーンアップに委譲
- `96f6379` worktree削除時に--forceオプションを追加
- `d128804` deploy.sh完了時にワークツリーを自動削除する
- `a01f6f0` deploy.shでワークツリーを自動削除しない（CWD無効化防止）

根本問題: ワークツリーを自動削除すると、Claude Codeのカレントディレクトリ（CWD）が無効になり、以降の全コマンドが失敗する。この副作用を見落として「自動削除追加→やめる」を繰り返した。

## Decision

- deploy.sh はワークツリーを自動削除しない
- deploy完了時にユーザーに手動削除コマンドを表示する
- CWDに影響する破壊的操作（ディレクトリ削除、移動）をスクリプト内で行わない

## Consequences

- メリット: CWD無効化によるコマンド全滅を防止
- デメリット: ワークツリーの手動クリーンアップが必要（`clean_gone` スキルで対応可能）
- deploy.sh に今後ワークツリー関連の変更を加える場合は、本ADRを参照すること
