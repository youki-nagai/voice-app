# 0002. 全サービスで動的ポートを使用する

Date: 2026-03-08

## Status

Accepted

## Context

deploy.sh内のサービス（PostgreSQL、E2Eテストサーバー、devサーバー）で固定ポートを使用していたため、ポート競合が繰り返し発生した:

**PostgreSQL（3回修正）:**
- `53cf403` PostgreSQL自動起動を追加、ポートを55432に変更
- `27dcada` PostgreSQLポートを動的割り当てに変更（ポート競合を完全回避）
- `d646e2e` deploy.sh Step8でもPostgreSQLを自動起動

**E2Eテストサーバー（3回修正）:**
- `2951029` E2Eテスト用サーバーを動的ポートで起動しワークツリーでの競合を解消
- `623db59` E2Eサーバーをtrapで確実に停止しプロセスリークを防止
- `188a3f3` ワークツリーのdevサーバーでポート8000を使わないように修正

根本問題: ワークツリーとメインリポジトリが同時に動作する環境で、固定ポートは必ず競合する。1つずつ動的ポートに変更したため3段階の修正が必要になった。

## Decision

- deploy.sh内で起動する全サービス（DB、テストサーバー、devサーバー）は動的ポートを使用する
- ポート番号はOSに割り当てさせ、環境変数で下流に渡す
- 固定ポートは本番環境（`make dev`）のみ許容する
- 新しいサービスをdeploy.shに追加する場合も動的ポートを必須とする

## Consequences

- メリット: ワークツリー並列実行時のポート競合を完全回避
- メリット: CI環境でも安全に実行可能
- デメリット: ポート番号が実行ごとに変わるためデバッグ時にログ確認が必要
- deploy.sh にサービスを追加する場合は、本ADRを参照すること
