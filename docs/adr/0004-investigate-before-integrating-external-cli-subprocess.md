# 0004. 外部CLIサブプロセス連携時の事前調査必須化

Date: 2026-03-08

## Status

Accepted

## Context

Claude CLIをサブプロセスとして呼び出す部分で5回の修正が発生した:

- `5287c4e` claudeコマンドのバイナリ探索を追加
- `c3690c4` stream-json出力に必須の--verboseフラグを追加
- `3046780` CLAUDECODE環境変数を除外してネストセッションエラーを回避
- `ad683d8` parse_response handles mixed text+JSON format from Claude
- `fd4287f` asyncio stdoutバッファ制限を1MBに拡大

根本問題: Claude CLIの動作仕様（バイナリパス、出力形式、環境変数の影響、バッファリング）を事前に調査せず、試行錯誤で1つずつ問題に対処した。外部CLIツールをサブプロセスとして組み込む際の調査不足が原因。

## Decision

外部CLIツールをサブプロセスとして組み込む際は、実装前に以下を調査・文書化する:

1. **バイナリパス**: 実行ファイルの配置場所、PATH解決、バージョン管理ツール（asdf等）での扱い
2. **出力形式**: stdout/stderrの形式（テキスト、JSON、混合）、フラグによる出力形式の変化
3. **環境変数**: 親プロセスから継承される環境変数の影響、除外すべき変数
4. **バッファリング**: stdoutのバッファリング挙動、ストリーミング時のフラッシュタイミング、バッファサイズ制限
5. **エラーケース**: 異常終了時の出力、タイムアウト、シグナルハンドリング

## Consequences

- メリット: 段階的な修正（5回→1回）を防止し、初回で正しく実装できる確率が上がる
- メリット: 調査結果がドキュメントとして残り、同じCLIツールの再利用時に参照できる
- デメリット: 初回実装時の調査コストが増加する
- 新しい外部CLIツールをサブプロセスとして組み込む場合は、本ADRを参照すること
